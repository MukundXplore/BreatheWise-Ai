## 🔴 Priority 1 — Integration (Must Have Before Demo)

### 1. Set Gemini API Key

Add your API key to the backend environment file to enable real AI-powered assessments instead of the mock engine.

**File:** `backend/.env`

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

**How to get:** [Google AI Studio → Get API Key](https://aistudio.google.com/app/apikey)

Once added, all `/api/assess` calls will use **Gemini 2.5 Flash** via the Gemini REST API.

---

### 2. Connect Elasticsearch (Optional but Recommended)

If you have Elastic Cloud credentials, enable real RAG retrieval by adding them to `.env`:

```env
ELASTIC_CLOUD_ID=your_elastic_cloud_id
ELASTIC_API_KEY=your_elastic_api_key
```

Then run the indexing script to populate the knowledge base:

```bash
cd backend
.venv\Scripts\activate
python ..\elastic\index_setup.py
```

> Without these credentials, the app seamlessly uses the **in-memory fallback** search engine with 11 pre-loaded guidelines — fully functional for the demo.

---

### 3. Run Both Servers

Use the provided helper script:

```powershell
cd "C:\Users\Farhan\Desktop\GDG Ai\BreatheWise"
.\run_servers.ps1
```

Or start them manually:

```bash
# Terminal 1 — Backend (Flask on port 8000)
cd backend
.venv\Scripts\activate
python -m flask --app app.main run --port 8000 --debug

# Terminal 2 — Frontend (React/Vite on port 5173)
cd frontend
npm run dev
```

App will be live at: **http://localhost:5173**

### 6. Live AQI Integration (Production Upgrade)

**Current state:** Uses WAQI API demo token → falls back to location-based simulation.

**For a real AQI feed:**

1. Register at [https://aqicn.org/api/](https://aqicn.org/api/) for a free token.
2. Add to `.env`:
   ```env
   WAQI_TOKEN=your_waqi_token_here
   ```
3. Update `fetch_delhi_aqi()` in `backend/app/main.py` to use the token.
